# エリア検索（ドリルダウンUX最適化）＆ 検索状態維持：実装計画書

## 1. 設計思想：シームレスな探索と検索状態の独立
- **状態の独立保持 (State Persistence)**: 「エリア」と「キーワード」を独立した検索軸として管理。キーワードがゼロになってもエリア選択（`area_id`）は破棄せず、広域検索として成立させる。
- **インクリメンタル・サーチの導入**: エリアを選択した瞬間（確定を待たず）に背景の店舗リストを更新し、ユーザーに即時フィードバックを返す。
- **暗黙的確定（Implicit Commit）**: 「決定」ボタンを押さずとも、枠外タップや階層の途中終了を「現在の選択肢での検索意思」とみなし、UXの摩擦を最小化する。
- **ステートフルな表示名**: 内部的な `area_id` と、ユーザーに提示する「表示名」を同期させ、ドリルダウン終了後も選択状態を可視化する。

## 3. 実装ロードマップ（更新順序）

### Phase 1: キーワードとエリアの疎結合化
1. **キーワード削除ロジックの修正**
   - キーワードチップの「×」ボタン押下時、現在のURLにある `area` パラメータを読み取り、それを引き継いだ状態でリクエスト（`/?area=xxx`）を投げるように変更。
   - すべてのキーワード（`q`）が空になっても、`area` があればエリア絞り込み状態でリストを表示し続ける。
2. **ナビゲーションの定義**
   - ロゴ/トップ：全パラメータ破棄（初期状態）。
   - エリア選択：既存キーワードを維持したまま `area` を追加/更新。

### Phase 2: リアルタイム絞り込みの連動
1. **`api/areaDrilldown.ts` のレスポンス拡張**
   - リスト項目に `hx-push-url="true"` を追加し、クリック時にURLを更新。この際、既存の `q` パラメータを消さないように制御。
   - `HX-Trigger` ヘッダーを使用して、店舗リスト（`#service-list`）を最新の `area` + `q` の組み合わせで再取得。

### Phase 3: 外側クリックによる「確定」と表示名の反映
1. **`SearchArea.tsx` の挙動修正**
   - 枠外タップ時、現在選択されている階層の `area_id` を確定値として採用し、検索を実行。
   - JSを用いて、ボタンの「エリアを選択」というテキストを選択した地域名（またはID）へ動的に書き換え。
