# エリア検索（ドリルダウンUX最適化）＆ 検索状態維持：実装計画書

## 1. 設計思想：シームレスな探索と検索状態の独立
- **状態の独立保持 (State Persistence)**: 「エリア」と「キーワード」を独立した検索軸として管理。キーワードが削除されてもエリア選択（`area_id`）は破棄せず、広域検索として成立させる。
- **階層ごとの意思決定（Granular Search）**: `SEARCH_EXECUTION_LEVEL`（現在は都道府県レベル）に基づき、ユーザーが探索の「区切り」に達したタイミングで画面全体をリロードし、検索結果を同期する。
- **暗黙的確定（Implicit Commit）**: 「決定」ボタンを廃止し、エリア名のクリックそのものを検索意思とみなすことで、UXの摩擦を最小化する。
- **ステートフルな表示名**: 選択された `area_id` に対応する名称をサーバーまたはURLパラメータから取得し、ドリルダウン終了後（リロード後）もボタンテキストとして可視化する。
