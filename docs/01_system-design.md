# 01_システム設計書（System Design）

## ファイル名: docs/01_system-design.md

## 1. 概要
本プロジェクトでは、HTMXによるサーバーサイドレンダリング（SSR）を軸としつつ、モダンなSPAに近い動的な検索体験を実現するために、**「State-on-HTML」**パターンを採用しています。特に「エリア検索」と「キーワード検索」の同期において、クライアント側で複雑な状態を持たず、DOMを唯一の真実（Single Source of Truth）として扱います。

---

## 2. コア・アーキテクチャ：State-on-HTML
UIの状態（現在の検索語句や選択エリアID）をJSのメモリ上ではなく、常に**「DOM（Hidden Input）」**に保持します。これにより、HTMXによる部分更新が行われても、常に最新の状態に基づいた再検索が可能になります。

### 主要なステートホルダー
- **`current-q-state`**: 正規化された検索キーワード（チップ化のソース）。
- **`current-area-state`**: 現在選択されている `area_id`。

---

## 3. 同期ロジックのシーケンス

### A. キーワード検索とチップ化（Chipify）
1. **ユーザー入力**: ヘッダーの検索窓から「小岩 小岩」等を入力。
2. **サーバー処理**: サーバー側（Hono）で重複削除等の正規化を行い、検索結果と共に `current-q-state` を返却。
3. **HTMX 部分更新**: `#search-result-module` が書き換わり、内部に新しい隠しフィールドが配置される。
4. **イベント検知**: `htmx:afterSettle` を JS が検知し、`current-q-state` の値を読み取る。
5. **DOM生成**: 
    - ヘッダー内の既存チップを全消去し、新しいチップ（`span` 等）を生成。
    - 各チップに「その単語を除外した再検索用URL」を `hx-get` として付与。
6. **再活性化**: `htmx.process()` を実行し、動的に生成された「チップ削除ボタン」をHTMXの動作対象として有効化。

### B. エリア・ドリルダウン検索
1. **トリガー**: `SearchArea.tsx` 内の `select` 要素で `change` イベントが発生。
2. **非同期リクエスト**: `hx-get="/api/areas?parent_id=..."` が走り、次の階層のリストを取得。
3. **URL同期**: `hx-push-url="true"` により、ブラウザのURLが `?area_id=13-123` のように書き換わる。
4. **結果表示**: 市区町村が選択されたタイミングで、自動的に `SearchResult.tsx` が更新される。

## 4. エリア・階層設計（v2.0 以降）

### 4.1 エリアID体系
親子関係をIDの文字列構造（ハイフン区切り・辞書順）で定義します。

| 階層 | ID例 | 属性・長さ |
|:---|:---|:---|
| 大エリア（地方） | `10` | 2文字 |
| 中エリア（都道府県） | `10-13` | 5文字 |
| バーチャルエリア | `10-01-V10` | 9文字（道央など） |
| 小エリア（市区町村） | `10-13-101` | 9文字 |

### 4.2 階層判定ロジック（LENGTH関数活用）
SQLおよびアプリケーション層では、`area_id` の文字数（LENGTH）を使用して階層を判定します。

~~~sql
-- 特定の親（13-123）の直下の子だけを取得するクエリ例
SELECT * FROM areas 
WHERE area_id LIKE '13-123-%' 
  AND LENGTH(area_id) = 13; -- 親の長さ + ハイフン + 3文字
~~~

## 5. ユーザー体験（UX）の最適化

### 5.1 検索窓の動的制御
- **プレースホルダー**: チップが存在する場合はプレースホルダーを非表示にし、チップが全消去された時のみ再表示します。
- **自動クリア**: キーワード確定後、入力欄を常に空にリセットし、連続した絞り込み（AND検索）を促します。

### 5.2 ローディング・インジケーター
- `hx-indicator` を使用し、データ取得中にスピナーや「⏳ 取得中...」のメッセージを表示。低速回線環境での不安を解消します。

---

## 6. 技術的制約と対策
- **ブラウザ戻るボタンの挙動**: `hx-push-url` を使用しているため、ブラウザの「戻る」で前の検索結果を再現可能ですが、その際にヘッダーのチップ表示も再同期させるため、`htmx:historyRestore` イベントでの再描画処理を実装します。
